Troubleshooting
Did I mention backups?
 

If you don't want to risk corrupting your world or make things even worse, make a manual backup first.

From the in game Mod Manager, click on "Open Mods Folder" and then exit the game. Go up one folder level and into the "Saves" folder. Identify your world by name and maybe datestamp, then copy it to a safe location.

Make sure that your game version matches your mod version.
 

On a mod page, the "Files" tab typically lists many different releases of a mod available for download.

It is important to know what game version you are running, and then to download the most appropriate mod version. As a general rule, major releases are compatible. So for game version 1.22.x, a mod that says "For Game verion" 1.22.x is going to work.

After making changes to your mod loadout, exit the game completely and clear the cache.
 

Exiting the game completely will ensure that memory is freed. You can also clear the cache at any time and this can fix a variety of issues.

From the in game Mod Manager, click on "Open Mods Folder" and then exit the game. Go up one folder level and into the "Cache" folder. Delete everything in there.

Clean up your mods.
 

Some strange things can happen when managing your loadout entirely from the in-game mod manager, so don't do that. It's generally ok for quick changes but things can go south in a hurry.

  1. From the in-game Mod Manager, click "Open Mods Folder". Exit the game.
  2. Ensure there's only one copy of each mod in the mods folder - delete all others.
  3. Since you are here, go up one folder level and go into the "Cache" folder. Delete everything inside that folder.
  4. Go up one folder level and go into the "ModConfig" folder. Delete the "primitivesurvival119.json" file inside that folder.
  5. Start the game, go back into the Mod Manager, and click the "Reload mods" button. Make sure that the mods are enabled.

Note: I never recommend disabling mods via the in-game Mod Manager - Open the Mods Folder and remove the disabled mod(s) completely from that folder.

Note 2: Don't unzip any mods. Just drop them in your mods folder zipped.

Missing or incorrect textures in game? This might be why.
 

If your game's texture atlases are too small all kinds of problems can crop up, from things in game having wrong textures to invisible blocks to the game crashing.

Pick ONE method below:

METHOD 1: Have the person that is experiencing the problem run these two commands in game (and then restart their game):

.clientconfig maxTextureAtlasWidth 8192
.clientconfig maxTextureAtlasHeight 8192


METHOD 2: Have the person that is experiencing the problem edit their %AppData%/Roaming/VintagestoryData/clientsettings.json file and change these two settings to something like:

"maxTextureAtlasWidth" : 8192,
"maxTextureAtlasHeight" : 8192,

It never hurts to reinstall the game.
 

On rare occasions, the game has become corrupted and needs to be reinstalled. Simply re-run the installer and reinstall it. Only takes a minute.

This is even more effective if you download and install the full game version from the client area.

https://account.vintagestory.at/

Review your log files.
 

The log files can reveal a lot about what's gone wrong.

From the in game Mod Manager, click on "Open Mods Folder" and then exit the game. Go up one folder level and into the "Logs" folder. The most important logs to review are client-main.log and server-main.log.

Focus on lines that start with [Fatal] or [Error] or [Warning]. They often also list the modid of the mod that is associated with the issue. Now in the case of warnings or errors, they don't necessarily mean that the mod in question is causing a problem, but ideally you don't want to see any such messages in your log files.

Suspect a mod conflict? Divide and conquer!
 

This is often also referred to as a binary search, and is often the quickest way to get to the root of such problems.

The wiki explains this procedure well enough. Just follow this link to read all about it.

https://wiki.vintagestory.at/Troubleshooting_Mods#Compatible_Mods

Performance issues? Lagging? Memory leaks?
 

These topics is well beyond the scope of this document. Below is some rambling from one of the game developers (RadFast) and myself. Good luck!

  • Do you have plenty of RAM? 32GB recommended on the client for heavily modded.
  • Do you have a modern CPU and GPU?

Let's try a few obvious things first.

  • Make sure you have no other software or apps running on the PC, except absolute essentials.
  • Check only one copy of Vintagestory.exe is running - use the Task Manager (Ctrl + Shift + Esc), list all processes in alphabetical order and check there is only one Vintagestory process.
  • If there is any exotic hardware connected (anything except mouse, keyboard, monitor, and speakers) disconnect it while testing.
  • Post a screenshot of your Graphics settings. Choose the "Medium" preset for testing, that should not stress the GPU at all. But click on "show all available settings" and choose "Max FPS: unlimited" and "VSync: off", while we are testing.
  • Check the PC is not overheating: if you built it yourself, are you sure CPU and GPU are well ventilated; make sure vents are not covered. Use a free tool like OpenHardwareMonitor to check the temperatures.
  • Reinstall the game, making sure to delete everything in the existing install folder (the Windows installer should automatically do it correctly).

If you still have the issue after trying the six things above, then next thing to try is: use the in-game performance monitor command .debug logticks 50. Play the game until the freezes start to occur. Then exit and post here the new client-main.log. What that command does is log in client-main.log any tick when the frame rate drops below 20 frames per second, together with detailed trace information which will make sense only to some people like the game devs.

Another helpful link:

https://wiki.vintagestory.at/Framerate_and_Performance
